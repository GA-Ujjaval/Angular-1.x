Number Scheme
Default Scheme allows Admin to define the Default Numbering Scheme for each object type viz. Parts, Products, Documents. The Scheme allows the Admin to define:
  • Prefix (optional for Parts; mandatory for Products and Documents when creating new default scheme)
  • Starting Number (required) - default value is 10000
  • Suffix (optional)
  • Increment by (required) - default value is 1
  • Running number - This displays the last part number that was auto-generated by the system. This value is read-only.
  • Test Next Numbers - allows you to view the next in the auto-generation sequence before applying changes.
With the above settings, the part numbers generated by the system would be 10000, 10001, 10002 and so forth.
NOTE: If the Setting in 'Revision Scheme' > 'Allow edits to object numbers, major and minor revisions' is ENABLED, users can edit the part number after it is auto-generated by the system. Let's assume the system auto-generated the part number 10002 and the user changed it to '90000', the next auto-generated number would be 10003. In other words, system always follows the auto-generation sequence defined by the values of 'starting number' and 'increment by' fields.
The Default Number scheme defined in the Parts tab, is used by default for Products and Documents objects as well.
To define a different Number scheme for Products and Documents, click on the 'Documents' tab > 'Create New default scheme for Documents'.
NOTE: Prefixes for each object type should be unique.
A company can easily define Part number settings as described:
  • Part Number scheme: Prefix = PT-; Starting Number = 100000; Increment by = 1….
    Numbers would be PT-100000, PT-100001,…
  • Document Number scheme: Prefix = DOC-; Starting Number = 100000; Increment by = 1….
    Numbers would be DOC-100000, DOC-100002,…
