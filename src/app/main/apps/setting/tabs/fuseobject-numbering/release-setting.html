Release Objects
Allow Manual Release of objects
NOTE: When enabled, objects can be released manually by going to the object > Change Status > 'Released', in addition to the workflow-based release process. When disabled, objects cannot be released manually and can be released only through the workflow-based release process.
Allow user to manually change status to 'InDevelopment'
NOTE: Only most recent revision of an object can be changed to 'InDevelopment'. When enabled, object's status can be manually changed to 'InDevelopment' by going to the Object page > Change Status > InDevelopment. If Disabled, object's status can be changed to 'InDevelopment', only through the workflow.
Allow Hierarchical Release of objects
NOTE: When enabled; if a user tries to 'release' an object manually or through the workflow, the entire hierarchy of that object (including all its child objects in the 'BOM' tab, and, document objects in the 'Attachments' tab), will be released.When disabled; if a user tries to 'release' an object manually or through the workflow, ONLY that object will be released (no child objects or document objects will be released).
Allow selection/unselection of objects during Heirarchical Release
NOTE: When enabled – user can select/unselect objects in the ‘hierarchical release’ dialog box.
When disabled, user cannot select/unselect any objects in the ‘hierarchical release’ dialog box.
