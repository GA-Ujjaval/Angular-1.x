Item Count setting

An item is any line displayed in the object tables. Items include object revisions. For eg: Part 123 RevA and Part 123 RevB are 2 different items.

Users can now choose the default number of items that are loaded in the objects tables from the database (parts/products/documents). This setting helps optimize the data-loading performance.

Set # of items to be loaded from database: 0 15 25 50 100 500 1000 1500 2000 All

NOTE: NOTE: You can change this setting at any time in the object table (in the header bar, under the total ‘Object count’ > “Items Loaded from database: 100”).

How this impacts Search and Filter:

If there are 1000 items in the database and the default number of items to load, is set to 100; Only the 100 most recently modified items will be displayed in the table.

‘Search’ parses through the entire database of 1000 items to display the results in the table.

However, the column ‘filters’ in the table, only filter results for data displayed in the table. In the above example, the column ‘filter’ will filter data from the 100 most recently modified objects.
