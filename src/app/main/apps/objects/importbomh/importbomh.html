Notes for importing a hierarchical (or multi-level or structured) BOM:

   Prerequisites:
  • 1) Hierarchical BOM Import is ONLY supported for CSV files, currently. All other file formats (including XLS and XLSX) will generate an error.
  • 2) Total number of records in the file cannot exceed 3000 (excluding the first row for title).
  • 3) The first row of the CSV file should contain the column titles.
  • 4) The file can contain only one tab.
  • 5) New attributes cannot be created in FusePLM on the fly, as part of the Import process. In other words, if the there’re attributes within the Excel file that you’d like imported, make sure to create those corresponding attributes in FusePLM prior to the import process.
  • 6) The Mandatory attributes, Part Number and Part Name, should not contain any formulas in the spreadsheet.
  • 7) There cannot be a blank column, in between columns with data. Eg: if column D is blank, but columns A, B, C and E have values in them, the file will not be imported. Delete the empty column and then re-import.
  • 8) The File should contain a column 'level', that represents the level of a particular part/sub-assembly in the multi-level BOM structure.
    • a. Import Manager will read the 'level' column from top to bottom.
    • b. The 'level' column in the file should start with 0. In other words, the second row in the spreadsheet should represent the top level assembly (level 0) into which the remaining BOM records are imported.
    • c. Values in the 'Level' column should be 0 or positive integers. Strings and float values are not permitted. It CANNOT be empty either.
    • d. There can be ONLY ONE row in the file with 'level' 0. If there are more than one rows with 'level' 0, then it’s an error condition and the import process will STOP.
  • 9) 'Broken' sequence in the 'Level' column, will cause an error condition. For example: The following sequence 0,1,1,1,2,2,7,2 for the level column, will result in an error. This is because a level 2 part should be followed by a level 3 part and not level 7 part. Import process will be STOPPED in this case. If multiple records with the same Part Number are found at the same level, ONLY the first record will be imported.

   Note:
  • 1) There is no limit on the number of levels that can be imported.
  • 2) FusePLM will read records in the file, until it encounters a blank row. All subsequent rows will be ignored.
  • 3) 'Import Hierarchical BOM' can be invoked ONLY from the 'Parts' module.
  • 4) If the record with 'level' 0 in the file, exists in FusePLM:
    • a. In 'Released'/'Obsolete' State, no child parts will be imported into it's BOM.
    • b. In 'InDevelopment' State, the matching FusePLM part's BOM, will be completely replaced by the child parts in the CSV file.
  • 5) If any sub-assembly or non-flat part in the file, exists in FusePLM:
    • a. In 'Released'/'Obsolete' State, the matching FusePLM part will be referenced in the BOM.
    • b. In 'InDevelopment' State, the matching FusePLM part's BOM, will be completely replaced by the child parts in the CSV file.
  • 6) If any flat record in the file, exists in FusePLM:
    • a. In 'Released', 'Obsolete' or 'InDevelopment' state, the existing FusePLM part will be referenced in the BOM.

Replace child parts of sub-assembly in FusePLM
NOTE:
  • When Disabled (default behavior), if any part in the file (flat part or sub-assembly), exists in FusePLM, the matching FusePLM part will be referenced as-is in the BOM (including any child parts that it may have in FusePLM).
  • When Enabled,
    • If any sub-assembly or non-flat part in the file, exists in FusePLM:
      • In 'InDevelopment' State, the matching FusePLM part's BOM, will be completely replaced by the child parts in the CSV file.
      • In 'Released'/'Obsolete' State, the matching FusePLM part will be referenced in the BOM as-is (unless, the setting System Settings > Released Setting > 'Allow edits to BOMs of released objects' is enabled)
    • If any flat record in the file, exists in FusePLM:
      • In 'Released', 'Obsolete' or 'InDevelopment' state, the matching FusePLM part will be referenced in the BOM.
